Mysql索引

索引可以大大提高MySQL的检索速度。

创建索引时，你需要确保该索引是应用在 SQL 查询语句的条件(一般作为 WHERE 子句的条件)。

实际上，索引也是一张表，该表保存了主键与索引字段，并指向实体表的记录。

  • 索引的优点：

    索引用来快速地寻找那些具有特定值的记录，如果作为搜索条件的列上已经创建了索引，MySQL无需扫描任何记录即可迅速得到目标记录所在的位置。 数据库索引好比是一本书前面的目录，能加快数据库的查询速度。

  • 索引的缺点：

    虽然索引大大提高了查询速度，同时却会降低更新表的速度，如对表进行INSERT、UPDATE和DELETE。因为更新表时，MySQL不仅要保存数据，还要保存一下索引文件。建立索引会占用磁盘空间的索引文件。一般情况这个问题不太严重，但如果你在一个大表上创建了多种组合索引，索引文件的会膨胀很快。索引只是提高效率的一个因素，如果你的MySQL有大数据量的表，就 需要花时间研究建立最优秀的索引，或优化查询语句。

创建普通索引：

最基本的索引，它没有任何限制，比如上文中为title字段创建的索引就是一个普通索引，MyIASM中默认的BTREE 类型的索引

1. 修改表结构的方式添加索引：
ALTER TABLE 表名 ADD INDEX 索引名  (字段(length))；length  长度

2. 创建索引：
Create index 索引名 on 表名（字段名）；

3. 创建索引（创表时）：
Create table 表名（
    字段 数据类型，
    Index 索引名 （字段）
）engine=InnoDB default charset=utf8；

4. 删除索引：
Drop index 索引名 on 表名；
Alter table 表名 drop index 索引名；

创建唯一索引：

与普通索引类似，不同的就是：索引列的值必须唯一，但允许有空值（注意和主键不同）。如果是组合索引，则列值的组合必须唯一，创建方法和普通索引类似

1. Alter table 表名 add unique [index] [索引名] （字段）；

2. Create unique index 索引名 on 表名（字段名）；

3. Create table 表名（
    字段 数据类型，
    Unique [index] 唯一索引名（字段）
）engine=InnoDB default charset=utf8；

4. 删除唯一索引：
Drop index 唯一索引名 on 表名；
Alter table 表名 drop index 索引名；

创建全文索引：

FULLTEXT索引仅可用于 MyISAM（引擎） 表 他们可以从CHAR、VARCHAR或TEXT列中作为CREATE TABLE语句的一部分被创建，或是随后使用ALTER TABLE 或CREATE INDEX被 添加。////对于较大的数据集，将你的资料输入一个没有FULLTEXT索引的表中，然后创建索引，其速度比把资料输入现 有FULLTEXT索引的速度更为快。不过切记对于大容量的数据表，生成全文索引是一个非常消耗时间非常消耗硬盘空间 的做法。

1. Alter table 表名 add  fulltext  [index] [索引名] （字段）；

2. Create fulltext [index] [索引名 on 表名（字段名）；

3. Create table 表名（
    字段 数据类型，
    Fulltext [index] [索引名] （字段）
）engine=MyISAM default charset=utf8；

4. 删除全文索引：
Drop index 索引名 on 表名；
Alter table 表名 drop index 索引名；

创建组合索引：

1. 修改表结构的方式添加索引：
ALTER TABLE 表名 ADD INDEX 索引名  (字段1，字段2.。。。)；length  长度

2. 创建索引：
Create index 索引名 on 表名（字段1，字段2.。。。）；

3. 创建索引（创表时）：
Create table 表名（
    字段 数据类型，
    Index 索引名 （字段1，字段2.。。。）
）engine=InnoDB default charset=utf8；

4. 删除索引：
Drop index 索引名 on 表名；
Alter table 表名 drop index 索引名；
