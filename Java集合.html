# Java 集合 #### 也就是用于存储数据的容器。 ##### 特点： 1. 对象封装数据，对象多了也需要存储。集合用于存储对象。 2. 对象的个数确定可以使用数组，但是不确定怎么办？可以用集合。因为集合是可变长度 的。 ##### 集合和数组的区别： 1. 数组是固定长度的；集合可变长度的。 2. 数组可以存储基本数据类型，也可以存储引用数据类型；集合只能存储引用数据类型。 3. 数组存储的元素必须是同一个数据类型；集合存储的对象可以是不同数据类型。 ### Collection 接口： ![image](http://opoh7f0qc.bkt.clouddn.com/JiHe.png) ##### Collection方法： Collection集合只能单个存储元素，并且只能存储引用类型 boolean add(object element)----->向集合中添加元素 void clear()-------------------->清空集合 boolean contains(Object o)------>判断集合中是否包含某个元素 注意：contains方法底层调用的是equals方法。如果equals返回true，就是包含。 存储在集合中的元素应该去重写equals方法 boolean isEmpty()--------------->判断集合中是否有元素 Iterator iterator()---------->获取集合所依赖的迭代器对象 boolean remove(Object o)-------->删除集合中某个元素 int size()---------------------->获取集合中元素的个数 Object[] toArray()-------------->将集合转换换成数组 #### List（接口）： **存储元素特点：** 1. 有序（List集合有下标）：存进去什么顺序，取出来的时候还是这个顺序。 2. 可重复的 **ArrayList：** 底层采用的是数组存储元素的，所以ArrayList适合查询，不适合频繁的随机增删元素。 **LinkedList：** 底层采用双向链表这种数据结构存储数据的，链表适合频繁的增删元素，不适合查询操作。 **Vector：** 底层和ArrayList集合相同，但是Vector是线程安全的。效率低，很少使用！ get(下标)------------>获取元素 #### Set（接口）： **存储元素特点：** 无序，不可重复。存进去的时候是一个顺序，取出来的时候就不一定是这个顺序 **HashSet（哈希表/散列表）：** > 1. HashSet底层实际上是一个hashMap，HashMap底层采用了哈希表数据结构。 > 2. 哈希表又叫做散列表，哈希表底层是一个数组，这个数组中每一个元素是一个单向链表。每一个单向链表都又一个独一无二的hash值，代表数组的下标。在某个单向链表中的每一个节点上的hsah值是相等的，hash值实际上是key调用hashCode方法，在通过"hash function"转换成的值 > 3. 如何向哈希表中添加元素： > 先调用被存储的key的hashCode方法，经过某个算法得出hash值，如果在这个哈希表中不存在这个hash值，则直接加入元素，如果该hash值已经存在。继续调用key之间的equals方法，如果equals方法返回false，则将该元素添加，如果equals返回true，则放弃添加该元素。 > 4. HashSet其实是HashMap中的key部分，HashSet有什么特点，HashMap中key应该具有相同的特点。 **SortedSet（接口）：** 存储元素特点： 无序，不可重复的。但是存储进去的元素可以按照元素的大小自动排序。 **TreeSet：** ##### 单向链表： > 每个元素都称作节点（Entry），每个节点都由两部分组成。 > 注意：每一个节点在内存中存储上在空间位置上是无规律的。 > 1. 为什么单向链表查询效率比较低？ > 单向链表中的每个元素在空间的存储位置上没有规律，也没有顺序，那么在查找某个元素的时候必须从头节点挨着往后找，直到找到为止。 > 2. 为什么单项链表的增删效率较高？ > 增删元素因为链表每个元素存储的空间是没有顺序的，删除或者添加某个元素，只需要让指针重新指向即可。不需要将其他元素位移。所以随机增删效率较高。 > **双向链表：**（是一个环状） 节点：三部分组成 ##### Map（接口）： Map和Collection没有关系， Map集合以键值对的方式存储元素 注意：键的特点：无序不可重复的 **HashMap：** 哈希表/散列表 HashMap中的Key等同于一个Set集合， **Hashtable：** 线程安全的效率低 Properties： 属性类，也是Key和Value的方式存储元素，但是Key和Value只能存储字符串 **SortedMap：** 存储元素特点： 无序，不可重复的。但是存储进去的元素可以按照元素的大小自动排序。 SortedMap中的Key等同于SortedSet **TreeMap：** TreeMap的Key就是一个TreeSet
